# ~={green}Summary=~

The system of organisation I'm using uses two folder systems. The first one being ACCESS and the second one being Johny.Decimal which is used mostly as inspiration not as a whole. The library is divided into folders based on access and each folder has numeral mark based on Johny decimal. 

---
# ~={green}Explanation=~

### ~={lightgreen}ACCESS=~
> folder system focused on clean structure 

***Folder system ACCESS is divided into 7 parts:***
0. *Encounters* -> unfinished files  
1. *Atlas* -> folder system organisation (mocs, bases, etc.)
2. *Calendar* -> any time related notes (daily, to-do, etc.)
3. *Cards* -> any short notes (quick notes, ideas, dictionary, etc.)
4. *Extras* -> any extra things to be kept (attachments, templates, etc.)
5. *Sources* -> notes based on external materials (analysis, refs, etc.)
6. *Spaces* -> big folders that are main parts of life (life, school, work, major hobbies, etc)

***Spaces are divided into:***
1. *MOCs* -> Maps Of Content (for navigation in the space)
2. *Areas* -> Bigger sections of the space (to keep big sections clean)
3. *Projects* -> any ongoing projects of the space
4. *Support Notes* -> anything else that doesn't belong anywhere
### ~={lightgreen}Johny.Decimal=~
>folder system that creates chaotic structure that has unbelievably clean navigation

Folder system that uses numbers to mark everything based on where the things belong without any folders at all.

### ~={lightgreen}Folder System Created=~
>system based on access that uses numbers for folders to make the navigation faster and easier

- uses access dividing 1 to 1 
- each folder has number but files don't
- heavy tagging implemented for organisation purposes
- templates implemented for faster writing
- subfolders added to make the division clear and visually appealing
- color coded for visual appeal

# ~={green}Spaces Tutorial=~
> I do recommend to sort the spaces in a way that you find most comfortable. If you are unsure there is no problem in using the folder system of mocs, areas, projects, support notes.

### ~={lightgreen}Areas=~

***Definition:*** Ongoing, long-term responsibilities or aspects of your life that need to be maintained over time.

***Think:*** “Always active,” not “done when complete.”

***Examples of Areas:***

- **Health** → Includes exercise routines, sleep tracking, doctor visit notes.
- **Relationships** → Reflections on friendships, family, connection habits.
- **Finances** → Budget overview, saving strategies, recurring bills. 
- **Career Development** → Goals for improvement, certifications, performance logs.
- **Personal Growth** → Journaling practices, skill learning paths (e.g., language learning).

***Use Areas to reflect maintenance:*** If it’s a “hat you wear” or a “role you play,” it belongs here.

### ~={lightgreen}Projects=~

***Definition:*** Specific, outcome-oriented efforts with a clear end goal and due date (or at least a done state).

**T*hink:*** “Has an end,” even if it’s far off.

***Examples of Projects:***

- **Redecorate Bedroom** → Tasks, budget, shopping list, mood board.
    
- **Plan 2025 Summer Vacation** → Itinerary, bookings, to-dos.
    
- **Write Short Story Collection** → Progress tracking, draft links, publishing goals.
    
- **Learn Python Basics** → Milestones, resources, completion checklist.
    

Projects end, Areas don’t. Once done, a project is archived. If it repeats often, consider turning it into an Area.

### ~={lightgreen}Support Notes=~

***Definition:*** Notes that support your Areas and Projects with research, reference material, or ideas.

***Think:*** “Library” or “raw material.”

***Examples of Support Notes:***

- A **guide** on Pomodoro Technique (used in your Productivity Area or Study Project)
- A **list of gift ideas** (used in Relationships or Birthday Project)
- A **reference** on tax deductions (used in Finances Area or Tax Filing Project)
- A **recipe** you want to try (used in Health Area or Meal Planning Project)

Support Notes don’t need structure or timeline. They exist to feed other files.
